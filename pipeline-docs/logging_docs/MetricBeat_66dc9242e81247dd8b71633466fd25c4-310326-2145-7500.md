## **MetricBeat** 

MetricBeat is a lightweight shipper that you can install on your servers to periodically collect metrics from the operating system and from services running on the server. MetricBeat takes the metrics and statistics that it collects and ships them to the output that you specify, such as Elasticsearch or Logstash. 

MetricBeat helps you monitor your servers by collecting metrics from the system and services running on the server, such as: 

Apache HAProxy MongoDB MySQL Nginx PostgreSQL Redis System Zookeeper 


![](logging_docs/markdown_converted/images/MetricBeat_66dc9242e81247dd8b71633466fd25c4-310326-2145-7500.pdf-0001-04.png)


**FIPS** Linux requires package `metricbeat-fips` , which is FIPScompliant. Windows package is not FIPS-compliant, but it has permanent exception. 

See Modules for the complete list of supported services for your version of MetricBeat.

## Additional fields 

In case you need to add some additional fields to your message which aren't present in the original message, you can achieve this like that: 

```
fields_under_root: true
fields:
```

```
  extra_fields:
          field_a: value_a
          field_b: value_b
```

```
metricbeat
  modules:
  - module: system
    metricsets:
      - cpu
      - load
      - memory
output:
  logstash:
    hosts: <FILL_ENV_ENDPOINT>:10301
    ssl:
       enabled: true
       verification_mode: none
```

```
logging:
  to_files: true
  files:
    path: /var/log/metricbeat
    rotateeverybytes: 10485760 # = 10MB
    keepfiles: 7
  level: info
```

## FIPS compliance 

There is a small change needed to be FIPS compliant 

```
output:
  logstash:
    hosts: <FILL_ENV_ENDPOINT>:10301
    ssl:
       enabled: true
       supported_protocols: [TLSv1.2]
       verification_mode: full
       certificate_authorities:
       - <PATH_TO_ENV_CERT>
```

The `certificate_authorities` is needed only when the OS CA certificates bundle doesn't contain the Root CA certificate used by the Logging Service.
