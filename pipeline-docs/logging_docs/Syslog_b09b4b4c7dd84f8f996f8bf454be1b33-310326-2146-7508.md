## **Syslog** 

Configure remote syslog from Unix/Linux and BSD/macOS to send logs to Papertrail. 

To log from a Unix system, edit the system's syslog daemon config file. These instructions are a reference. Papertrail will provide more specific instructions (including a log destination) when you add a system.

## **FIPS** 

The traffic is not encrypted and therefore it's not FIPScompliant 

These instructions will typically pick up operating system logs. Some apps write logs directly to text files, bypassing the syslog daemon. To collect logs from these apps, use remote_syslog2 instead.

## **Shipper extra fields** 

`message host` - connection originator's IP `facility` - one of: `kern, user, mail, daemon, auth, syslog, lpr, news, uucp, cron, authpriv, ftp, ntp, audit, alert, at, local0, local1, local2, local3,local4, local5, local6, local7 facility_code` - numerical code `0 -23 severity` - one of: `emerg, alert, crit, err, warning, notice, info, debug severity_code -` numerical code `0-7 logsource` - hostname from a message `program pid` - PID of program 

## Configuration 

Use **RFC3164** with support for ISO8601 date style Use TCP instead UDP since UDP  is unreliable per design and unusable in Concur network.
